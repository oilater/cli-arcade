# CLI Arcade 발표 대본

## 1. 프로젝트 소개

여러분 혹시 클로드에게 작업시키고 기다릴 때 심심하신 분 계신가요?

안녕하세요. 터미널에서 플레이하는 아케이드 게임 플랫폼, **CLI Arcade**를 소개하겠습니다.

저는 언젠간 게임을 직접 만들어봐야겠다는 생각을 갖고 있었는데요,
제가 크레이지 아케이드를 좀 잘합니다. 꼭 한번 만들어보고 싶었는데, 오늘 마라톤에서 드디어 만들게 되었습니다!

## 2. 게임 모드

Bomb Game은 세 가지 모드를 지원합니다.

- **솔로 모드** — 혼자서 봇 3마리와 대전
- **로컬 2P** — 한 키보드로 두 명이 동시 플레이
- **온라인 멀티** — WebSocket 기반 네트워크 대전

---

## 3. 상태 관리 — 불변성 패턴

게임 구현에서 가장 중요하게 생각한 설계 원칙은 **완전한 불변성**입니다.

모든 상태 변경은 기존 객체를 직접 수정하지 않고, 항상 새로운 객체를 반환하는 순수 함수로 구현했습니다.

```typescript
function tick(state) → 새로운 state
function movePlayer(state, ...) → 새로운 state
function placeBomb(state, ...) → 새로운 state
```

이렇게 하면 상태 추적이 명확하고, React의 렌더링 사이클과도 자연스럽게 연동됩니다.

게임 상태는 크게 6가지로 구성됩니다 — 맵 타일, 플레이어, 폭탄, 폭발, 다트, 아이템입니다.

---

## 4. 게임 루프

게임은 **100ms 간격**, 초당 10틱으로 동작합니다.

매 틱마다 순서대로 처리합니다:

1. 봇 AI 행동 결정 (솔로모드)
2. `tick()` 함수로 전체 게임 상태 갱신
3. 게임오버 판정

여기서 중요한 점은, 플레이어의 키 입력은 틱과 별개로 **즉시 상태에 반영**한다는 것입니다.
처음에는 입력을 틱에 모아서 처리했는데, 그러면 입력이 씹히는 문제가 있어서 즉시 반영 방식으로 바꿨습니다.

---

## 5. tick() — 핵심 게임 로직

`tick()` 함수가 게임의 심장입니다. 한 틱에서 5단계를 순서대로 처리합니다.

**1단계: 다트 이동**
- 틱당 2칸 이동하되, 1칸씩 두 번에 걸쳐 이동합니다.
- 매 칸마다 충돌을 체크하고, 폭탄에 맞으면 해당 폭탄을 즉시 폭발시킵니다.

**2단계: 폭탄 타이머 감소**
- 모든 폭탄의 타이머를 1씩 줄이고, 0이 되면 폭발 큐에 넣습니다.

**3단계: 폭발 처리**
- 큐 기반으로 연쇄 폭발을 처리합니다.
- 폭탄이 터지면 4방향으로 범위만큼 확산되고, 블록을 파괴하면 40% 확률로 아이템이 드롭됩니다.
- 폭발 범위 안에 다른 폭탄이 있으면 큐에 추가되어 연쇄 폭발이 일어납니다.

**4단계: 사망 판정**
- 폭발 위치에 있는 플레이어가 사망합니다.
- 솔로모드에서는 봇이 인간 플레이어의 폭발에만 죽도록 `owner` 필드로 구분했습니다.

**5단계: 승리 판정**
- 생존자가 1명 이하이면 게임이 종료됩니다.

## 6. 봇 AI — 규칙 기반 + BFS

봇은 딥러닝 같은 학습 기반이 아니라, **규칙 기반 + BFS 탐색** AI입니다.

매 틱마다 우선순위에 따라 행동을 결정합니다:

1. 위험하면 → **BFS로 탈출 경로를 탐색**해서 이동
2. 안전하면 → **탈출로가 2개 이상**일 때만 폭탄 설치
3. 근처에 아이템이 있으면 → 아이템 쪽으로 이동
4. 블록 옆이면 → 폭탄 설치
5. 해당 없으면 → 랜덤 이동

핵심 함수는 세 가지입니다:
- `isDangerous()` — 폭발 범위인지, 쿨다운 중인지 판단
- `findEscapeDir()` — BFS로 최대 15칸까지 안전한 칸을 탐색
- `countEscapeRoutes()` — 폭탄 설치 전 탈출로가 충분한지 확인

## 7. 렌더링

렌더링은 `BombGrid` 컴포넌트에서 담당합니다.

셀마다 **우선순위**에 따라 표시 내용을 결정합니다:
폭발 > 다트 > 플레이어 > 폭탄 > 아이템 > 지형 순서입니다.

각 레이어를 `useMemo`로 Map에 캐싱해서 **O(1)로 조회**하고, 매 틱마다 전체 그리드를 다시 그립니다.

죽은 플레이어도 회색 X로 표시해서 어디서 죽었는지 알 수 있고,
살아있는 플레이어가 위를 지나갈 때는 살아있는 쪽이 우선 표시됩니다.

## 8. 멀티플레이어

온라인 모드는 **클라이언트-서버 구조**입니다.

**서버 (호스트)**
- Bun 내장 WebSocket 서버를 사용합니다.
- 서버가 게임 루프를 돌리고, 매 틱마다 전체 상태를 브로드캐스트합니다.
- 서버가 권위 있는(authoritative) 상태를 가집니다.

**클라이언트**
- 키 입력을 서버로 전송하고, 서버에서 받은 상태를 그대로 렌더링합니다.
- 연결이 끊어지면 자동으로 재접속합니다.

같은 네트워크에서 호스트의 LAN IP로 접속하면 바로 대전할 수 있습니다.

## 9. 개발 중 해결한 문제들

개발하면서 몇 가지 흥미로운 문제를 만났습니다.

**봇이 자폭하는 문제** — 봇이 자기 폭발에 죽어서 게임이 바로 끝났습니다. Explosion에 `owner` 필드를 추가해서, 솔로모드에서 봇은 인간의 폭발에만 죽도록 했습니다.

**다트가 폭탄을 건너뛰는 문제** — 한 번에 2칸을 점프시키니까 중간 폭탄을 놓쳤습니다. 1칸씩 두 번 이동하면서 매 칸 충돌 체크하도록 수정했습니다.

**키 입력이 씹히는 문제** — tick이 상태를 덮어써서 입력이 사라졌습니다. 입력을 즉시 상태에 반영하는 방식으로 해결했습니다.

**멀티에서 폭탄에 안 죽는 문제** — 솔로용 봇 면역 로직이 온라인에도 적용되고 있었습니다. 파라미터를 optional로 바꿔서 모드별로 분리했습니다.

**WebSocket 연결 실패** — `localhost`가 IPv6로 해석되면서 연결이 안 됐습니다. `127.0.0.1`을 명시하고, 서버는 `0.0.0.0`으로 바인딩해서 해결했습니다.
